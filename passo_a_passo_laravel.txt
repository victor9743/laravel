CRIANDO CONTROLLER

php artisan make:controller NomeController

CRIANDO VIEW

php artisan make:view main NomeView

se eu quiser colocar dentro de um diretório

php artisan make:view main dir/NomeView


CRIANDO MIGRATES
php artisan make:migration create_users_table

// Exclusão lógica: marca o registro como apagado usando a coluna deleted_at,
// sem remover o registro fisicamente do banco de dados
$table->softDeletes();

No model:

use Illuminate\Database\Eloquent\SoftDeletes;

class User extends Model
{
    use SoftDeletes;
}


Sem isso, o Laravel não aplica o soft delete automaticamente.

executar as migrations

php artisan migrate

executar o rollback
php artisan migrate:rollback


criando seeder
php artisan make:seeder UsersTableSeeder

executa seeder
php artisan db:seed NomedaSeed

CRIANDO MODEL 

php artisan make:model NomeDoModel


TRANSFORMA OS DADOS VINDO DO ELOQUENT MODEL EM ARRAY

$users = User::all()->toArray();

REDIRECIONA PARA A PÁGINA ANTERIOR: ->back()
REDIRECIONA REPASSANDO OS DADOS QUE FORAM ENVIADOS: ->withInput()

return redirect()->back()->withInput()->with('loginError', 'Username ou Password Inválidos');

USANDO SOFT DELETE

EXEMPLO NO MODEL:
<?php

namespace App\Models;

use Illuminate\Database\Eloquent\Model;
use Illuminate\Database\Eloquent\SoftDeletes;

class Note extends Model
{
    use SoftDeletes;
    
    public function User(){
        return $this->belongsTo(User::class);
    }
}

TRABALHANDO COM AUTENTICAÇÃO COM BREEZE

1- INSTALAÇÃO
composer require laravel/breeze --dev
php artisan breeze:install

SE HOUVER ERRO, REALIZAR ESTES PASSOS
Apague tudo que o Composer gera:

rm -rf vendor composer.lock


Limpe o cache do Composer:

composer clear-cache


Reinstale as dependências:

composer install


Depois disso, instale o Breeze novamente:

composer require laravel/breeze --dev
php artisan breeze:install
